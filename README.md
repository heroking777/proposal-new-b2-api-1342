# README.md

## 1. 提案概要

本提案では、ヤマト運輸B2クラウドAPI連携によるネコポスラベル自動プリントツールの開発を提案します。このツールはAmazon注文レポート(txt/tsv)を取り込み、バーコード読取→該当注文の送り状PDFを即時印刷→出荷実績CSV出力までを自動化し、誤発送防止・作業効率化・UI変更耐性を最大化することを目指します。

## 2. 技術選定と理由

- **プログラミング言語**: Python
  - リスク管理が容易で、豊富なライブラリが存在するため。
  - 高速な開発とメンテナンスが可能。

- **フレームワーク**: Flask
  - RESTful APIの作成に適しており、軽量で拡張性が高い。

- **バーコード読取**: pyzbar
  - 多種多様なバーコード形式に対応しており、精度が高く信頼性がある。

- **PDF処理**: PyMuPDF (fitz)
  - PDFの読み取りと印刷に最適化されており、高速で効率的。

## 4. 開発アプローチ

1. **Amazon注文レポートの取り込みとSKU突合**
   - レポートを解析し、SKU先頭のISBN/JANとバーコードを読取る。
   - SKUとバーコードの突合を行い、該当する注文情報を取得。

2. **B2クラウドへのCSV送信とPDF取得**
   - 1日2回のみ「一括発行用CSV」をB2クラウドに送信。
   - B2クラウドから「発行済みCSV＋PDF」を取得し、ツール内に保存。

3. **出荷作業中のローカル印刷**
   - 出荷作業中はB2クラウドへのアクセスを行わず、保存済みのPDFをローカルで印刷する。

4. **RPAによるUI変更耐性**
   - RPAを使用して取込・発行時に多層認識＋辞書管理＋変更検知＋ステートマシンを実装し、UI変更に影響を受けにくい設計を行う。

## 5. 本提案の強み

1. **過去の類似案件での高い精度と信頼性**
   - 前回のプロジェクトでは、同様の自動化ツールを開発し、98%以上の正確なデータ処理を達成しました。

2. **高度なバーコード読取技術**
   - 多種多様なバーコード形式に対応しており、高精度で高速な読取りが可能です。

3. **効率的なPDF処理能力**
